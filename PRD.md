# 폼 빌더 (백엔드 불필요) — PRD

## 1. 제품 개요

**제품명**: 폼 빌더 (Form Builder)  
**핵심 가치**: Typeform·JotForm 같은 SaaS 없이, 드래그&드롭으로 폼을 만들고 응답을 로컬 SQLite에 저장하는 **무료, 로컬-우선 폼 빌더**.

**대상 고객**: 소규모 사업 (폼 몇 개만 필요하고 로컬 저장 선호)
- 카페·식당·온라인숍 운영자
- 프리랜서·소규모 서비스 사업가
- 개인 블로거·크리에이터

**차별점**:
- ✅ **로컬 완결**: 클라우드 전송 없음, 데이터는 자기 컴퓨터에 저장
- ✅ **무료 기본**: 월 구독료 없음
- ✅ **간단함**: UI 극대화 (Typeform 대비 40% 더 간단)
- ✅ **정산 투명**: 응답 수 제한 없음

**제약사항** (명시적으로 문서화):
- 클라우드 백업 불가 (MVP)
- 팀 협업 불가 (MVP)
- 응답 알림 불가 (MVP)
- Google Drive 연동 불가 (나중)

---

## 2. 타겟 페르소나

### Primary: 소규모 사업주 (20~55세)
- **Pain Point**: Typeform 월 $25+ 너무 비쌈, 클라우드에 고객 데이터 올리기 꺼려짐
- **Use Case**: 예약 폼, 설문조사, 고객 피드백, 이메일 수집
- **Tech Literacy**: 낮음 (드래그&드롭, 클릭만 가능)
- **Value**: 무료 + 빨리 만들기

### Secondary: 개발자/스타트업 (1~3명)
- **Pain Point**: 커스텀 폼 코딩하기 싫음, 빨리 프로토타입 필요
- **Use Case**: 베타 테스트 폼, 알파 사용자 모집
- **Tech Literacy**: 높음 (API, JSON 이해)
- **Value**: 빠른 구축 + 로컬 저장 + CSV 내보내기

## 3. 핵심 기능 (Must-Have)

### 3.1 폼 빌더 편집기 ⭐
**기본**: 드래그&드롭으로 필드 추가/제거
- **서브기능 1**: 필드 유형 6종 이상 지원
  - 단답 (Single Line Text)
  - 장문 (Long Text / Paragraph)
  - 객관식 (Multiple Choice)
  - 체크박스 (Checkboxes)
  - 드롭다운 (Dropdown)
  - 평점 (Rating Scale, 1~5 ⭐)
  - *선택 필드 표시 (Required)*
- **서브기능 2**: 필드 순서 변경 (드래그)
- **서브기능 3**: 필드 설정 (레이블, 플레이스홀더, 기본값, 필수 여부)

### 3.2 폼 공개 링크 & 응답 수집 ⭐
**기본**: 폼 저장 후 공개 URL 생성
- **서브기능 1**: 응답자가 폼 링크로 접근 → 응답 입력 → 저장 (DB 저장)
- **서브기능 2**: 응답 받을 때마다 타임스탬프 기록
- **서브기능 3**: 동일한 응답자 여러 번 제출 가능

### 3.3 응답 목록 (List + 검색/필터) ⭐
**기본**: 받은 응답을 표로 표시
- **서브기능 1**: 응답 테이블 조회 (모든 필드 표시)
- **서브기능 2**: 검색 (자유 텍스트 검색, 단답/장문 필드만)
- **서브기능 3**: 필터 (필드별 필터, 예: "객관식 옵션 A 선택한 응답만")

### 3.4 응답 통계 & 대시보드 ⭐
**기본**: 응답 분석 화면 ("대시보드" 명칭)
- **서브기능 1**: 응답 총 개수 표시
- **서브기능 2**: 객관식/체크박스/드롭다운: 옵션별 응답 분포 (카운트 + 원형 차트)
- **서브기능 3**: 평점 필드: 평균 평점 + 분포 차트

### 3.5 CSV 내보내기 ⭐
**기본**: 응답을 CSV 파일로 다운로드
- **서브기능 1**: 모든 응답 + 모든 필드 포함
- **서브기능 2**: 폼 생성일 기준 "폼명_응답_yyyyMMdd.csv" 자동 이름
- **서브기능 3**: 한글 인코딩 (UTF-8 BOM)

### 3.6 빈 상태 (Empty State) 처리 ⭐
**기본**: 폼 없을 때, 응답 없을 때, 필터로 응답 0일 때 UI
- **서브기능 1**: "아직 만든 폼이 없습니다" 메시지 + "새 폼 만들기" 버튼
- **서브기능 2**: 응답 없을 때: "아직 받은 응답이 없습니다"
- **서브기능 3**: 필터로 응답 0일 때: "조건에 맞는 응답이 없습니다"

## 5. 화면 목록 (7개+)

| # | 화면명 | 경로 | 설명 |
|---|--------|------|------|
| 1 | **홈** | `/` | 폼 목록 + "새 폼 만들기" 버튼 |
| 2 | **폼 편집** | `/forms/[id]/edit` | 드래그&드롭 폼 빌더 |
| 3 | **폼 응답 (공개)** | `/f/[publicId]` | 응답자가 접근하는 폼 페이지 |
| 4 | **응답 목록** | `/forms/[id]/responses` | 응답 테이블 + 검색/필터 |
| 5 | **응답 대시보드** | `/forms/[id]/dashboard` | 통계 차트 + 응답 총 개수 |
| 6 | **폼 설정** | `/forms/[id]/settings` | 폼 이름, 설명, 공개 링크 복사 |
| 7 | **응답 상세** | `/forms/[id]/responses/[responseId]` | 단일 응답 상세 보기 (선택) |
| 8 | **응답 검색/필터** | `/forms/[id]/responses?search=...&filter=...` | URL 쿼리로 상태 유지 |

## 7. 데이터 모델

### Table: forms
```
id          TEXT PRIMARY KEY       # UUID
name        TEXT NOT NULL          # 폼 이름
description TEXT                   # 폼 설명
publicId    TEXT UNIQUE NOT NULL   # 공개 폼 링크용 short ID
fields      JSON NOT NULL          # 필드 배열 [{id, type, label, options, required, ...}]
isPublished BOOLEAN DEFAULT true    # 공개 여부
createdAt   TIMESTAMP               # 생성일
updatedAt   TIMESTAMP               # 수정일
```

### Table: responses
```
id          TEXT PRIMARY KEY       # UUID
formId      TEXT NOT NULL          # forms.id FK
data        JSON NOT NULL          # {fieldId: value, ...}
userAgent   TEXT                   # User-Agent (응답자 기기 정보)
createdAt   TIMESTAMP               # 응답 시간
```

### Table: fields (선택: 정규화)
또는 fields를 JSON 배열로 forms.fields에 직렬화.

## 8. 기술 스택

| 계층 | 기술 |
|------|------|
| **프론트엔드** | Next.js 16 (App Router) + TypeScript |
| **UI 컴포넌트** | shadcn/ui |
| **드래그&드롭** | react-beautiful-dnd 또는 dnd-kit |
| **차트** | Recharts (응답 분포 원형/막대 차트) |
| **폼 검증** | zod |
| **상태 관리** | React Context / zustand (선택) |
| **데이터베이스** | SQLite (better-sqlite3) |
| **ORM** | Drizzle ORM |
| **API 라우트** | Next.js API Routes |

## 9. 비기능 요구사항

| 항목 | 요구사항 |
|------|---------|
| **성능** | 폼 로드 < 1초, 응답 제출 < 500ms |
| **보안** | CSRF 토큰, XSS 방지 (sanitize), 입력 검증 |
| **신뢰성** | 응답 중 새로고침 → 저장 안 됨 (재입력 안내), 로컬 데이터만 |
| **접근성** | 스크린 리더 지원 (aria-label), 키보드 네비게이션 |
| **확장성** | 향후 다중 페이지 폼, 조건부 필드 지원 가능하도록 구조 |
